# 🐍 Το Παιχνίδι του Φιδιού 2025

Ένα παιχνίδι Snake που έχουμε φτιάξει από την αρχή σε C. Το φίδι κινείται στην οθόνη, τρώει το φαγητό και μεγαλώνει. Αν συγκρουστεί με τον εαυτό του ή τα τοιχώματα, τελειώνει το παιχνίδι!

## Πώς Είναι Χτισμένο

Το παιχνίδι έχει τρία κύρια μέρη που δουλεύουν μαζί:

```
┌──────────────────────┐
│  Το Κεντρικό Loop    │ ← Ελέγχει τα πάντα
│  (game.c)            │
├──────────────────────┤
│  Η Λογική Παιχνιδιού │ ← Υπολογισμοί και κανόνες
│  (state.c)           │
├──────────────────────┤
│  Γραφικά & Ήχος      │ ← Αυτό που βλέπεις/ακούς
│  (interface.c)       │
└──────────────────────┘
```

**Η Λογική Παιχνιδιού** αποθηκεύει:
- Πού είναι το φίδι ακριβώς
- Πού είναι το φαγητό
- Πόσα σημεία έχεις κερδίσει
- Αν έχει τελειώσει το παιχνίδι

**Τα Γραφικά και ο Ήχος** κάνουν:
- Ζωγραφίζουν το φίδι, το φαγητό και τη βαθμολογία
- Παίζουν μουσική φόντου
- Κάνουν ηχητικά εφέ όταν τρας φαγητό ή τελειώνει το παιχνίδι

**Το Κεντρικό Loop** κάνει:
- 60 φορές το δευτερόλεπτο ανανεώνει όλα
- Ελέγχει τι πάτησε ο παίκτης
- Ενημερώνει την κατάσταση
- Ζωγραφίζει την οθόνη

## Χρειάζεται Τι;

- **macOS** 
- **GCC** (μεταγλωττιστής)
- **raylib** - φτιάχνεται ήχος και γραφικά (`brew install raylib`)

## Πώς το Τρέχεις

```bash
make run
```

Αυτό κάνει κατασκευή και βγάζει το παιχνίδι αμέσως.

## Παίξιμο

- **Βέλη** ή **WASD** → Κινήσου
- **Space** → Παύση
- **ESC** → Έξοδος

Φάε το κόκκινο σημείο! Για κάθε φαγητό που τρας, μεγαλώνεις. Μην συγκρουστείς με τον εαυτό σου.

## Εντολές Κατασκευής

```bash
make              # Φτιάχνει το παιχνίδι
make run          # Φτιάχνει και τρέχει
make clean        # Σβήνει τα κατασκευασμένα αρχεία
```

## Τι Γίνεται Κατά την Κατασκευή

Ο μεταγλωττιστής παίρνει τα αρχεία C και τα κάνει εκτελέσιμο:

```
game.c → game.o (αρχείο αντικειμένου)
state.c → state.o
interface.c → interface.o
   ↓
Όλα μαζί → game (το τελικό παιχνίδι)
```

Κάθε αρχείο αντικειμένου είναι περίπου 2-6 KB, και το τελικό παιχνίδι είναι 41 KB.

## Η Λογική Κάθε Καρέ

1. Διαβάζουμε τα πλήκτρα
2. Κινούμε το φίδι
3. Ελέγχουμε αν έφαγε φαγητό
4. Αν ναι, μεγαλώνει
5. Ελέγχουμε αν συγκρούστηκε
6. Ζωγραφίζουμε όλα

Αυτό γίνεται 60 φορές το δευτερόλεπτο.

## Ανίχνευση Σύγκρουσης

- **Το Φαγητό:** Κοιτάζουμε αν το κεφάλι του φιδιού αγγίζει τον κύκλο του φαγητού
- **Αυτό-Σύγκρουση:** Κοιτάζουμε αν το κεφάλι φιδιού είναι σε άλλο σημείο του σώματος
- **Τοίχοι:** Κοιτάζουμε αν βγήκε έξω από την οθόνη

## Γραφικά και Ήχος

Κάθε καρέ κάνουμε:

1. Σβήνουμε την οθόνη (μαύρο φόντο)
2. Ζωγραφίζουμε κάθε κομμάτι του φιδιού (ορθογώνια)
3. Ζωγραφίζουμε το φαγητό (κύκλος)
4. Γράφουμε τη βαθμολογία
5. Ενημερώνουμε τη μουσική (παίζει στο background)
6. Δείχνουμε όλα

Η μουσική παίζει μόνες της όσο παίζεις. Όταν τρας φαγητό ή τελειώσει το παιχνίδι, ακούς έναν ήχο.

## Πόσο Δρομολόγιο Χρειάζεται

- **CPU:** Λιγότερο από 5% της ισχύος (το παιχνίδι δεν κόβει)
- **Μνήμη:** Περίπου 2-3 MB
- **Χρόνος Κατασκευής:** 2-3 δευτερόλεπτα για κάθε φορά
- **Κάθε Καρέ:** Περίπου 16 χιλιοστά του δευτερόλεπτου

## Αν Θέλεις να Προσθέσεις Κάτι

**Νέα Γραφικά ή Εικόνες:**
1. Βάλε την εικόνα στο `game/assets/`
2. Φόρτωσέ την στο `interface.c`
3. Ζωγραφίστη όπου θέλεις

**Νέα Ηχητικά Εφέ:**
1. Βάλε το αρχείο ήχου στο `game/assets/`
2. Φόρτωσέ το στο `interface.c`
3. Παίξε το όταν χρειάζεται

**Νέα Μηχανική Παιχνιδιού:**
1. Άλλαξε τη δομή δεδομένων στο `state.h`
2. Ενημέρωσε τη λογική στο `state.c`
3. Ζωγραφίστη όπου χρειάζεται στο `interface.c`

## Αποσφαλμάτωση

Αν υπάρχει πρόβλημα:

**"raylib.h not found"**
- Εγκατάστησε raylib: `brew install raylib`

**"Undefined reference to raylib"**
- Ελέγχει αν η διαδρομή `/opt/homebrew/Cellar/raylib/5.5/lib` υπάρχει

**Δεν ακούω ήχο**
- Δες αν τα αρχεία ήχου υπάρχουν στο `game/assets/`
- Δοκίμασε τον ήχο του συστήματος με κάτι άλλο
